# Issue: issue_crafter スキルの手順遵守強化

## 概要
`issue_crafter` スキルが定義されたワークフロー（Drafting -> Validation -> Planning -> Integration）を無視し、検証や計画なしに実装へ進もうとする問題を修正する。

## 背景と問題点
現在の `SKILL.md` では、各ステップの実行強制力が弱く、モデルが手順をスキップしてしまう。これにより、技術的な裏付けのないまま Issue が作成され、品質低下や手戻りの原因となる。

## 解決策
`skills/issue_crafter/SKILL.md` を修正し、以下の制約を追加する。
1.  **段階的実行の強制**: "STOP after each step and ask for user confirmation." を明記。
2.  **検証の義務化**: "Validation with `codebase_investigator` is MANDATORY." と記述。
3.  **進捗報告**: 各レスポンスに進捗状況を含めるよう指示。

## 実装内容詳細
### `skills/issue_crafter/SKILL.md` の改訂
1. **Workflow セクションの再定義**:
   - 各ステップに「STOP and wait for user confirmation before proceeding to the next step.」という指示を明記する。
   - **Validation** ステップにおいて、`codebase_investigator` の使用を「MANDATORY」とし、検証結果の報告と承認なしでの進行を禁止する。
   - **Integration** ステップの完了後、ユーザーに最終レビューを求めるよう指示する。

2. **Rules of Engagement セクションの新設**:
   - **Sequential Execution**: 一度に複数のステップを実行すること、またはステップをスキップすることを厳禁する。
   - **Explicit Confirmation**: ユーザーからの明確な "Next", "Go", またはそれに類する承認があるまで、次のフェーズへ移行してはならない。
   - **Evidence-Based**: すべての設計判断は `codebase_investigator` による調査結果に基づく必要がある。

3. **Status Reporting の義務化**:
   - 各レスポンスの末尾に、現在の進捗状況を以下の形式で表示させることを指示する。
     `Current Step: [Step Name] | Next Step: [Next Step Name] | Status: [Waiting for Approval / In Progress]`

## ゴール
- `issue_crafter` スキルが自律的に先走りせず、各ステップごとにユーザーの制御下で精密な Issue 作成プロセスを遂行できるようになること。
- スキル使用時に、必ず調査・検証フェーズが挟まれ、技術的な裏付けのある Issue が作成されること。
