# Issue: サブエージェント `simplifier` の `balancer` への改名と役割の再定義

## 概要
サブエージェント `simplifier` の名称を `balancer` に変更し、その役割を「単なる簡素化」から、要件と実装の複雑さの均衡を保つ「調停者・バランサー」へと再定義する。これにより、タイピングの利便性向上と、より実態に即した専門性の提供を図る。

## 背景と問題点
- `simplifier` という名称はタイピングが煩雑であり、開発効率を下げている。
- このエージェントの本質は、単にコードを削ることではなく、プロジェクトの真の要件と設計の複雑さの「均衡（バランス）」を保つことにあり、現在の名称は一部の役割しか表現できていない。

## 解決策
- `agents/simplifier.md` を `agents/balancer.md` にリネームする。
- エージェントの `name` および `description` を `balancer` に更新する。
- システムプロンプト内のペルソナ記述を、「単純化の専門家」から「均衡を保つ調停者（Balancer/Mediator）」へとアップデートし、オーバーエンジニアリングの排除と真の要件の充足のバランスを強調する。

## 実装内容詳細
1. **ファイルのリネーム**: `agents/simplifier.md` -> `agents/balancer.md`
2. **マニフェスト更新**:
   - フロントマターの `name` を `balancer` に変更。
   - `description` を「要件と複雑さの均衡を保つ調停者」等に更新。
3. **プロンプトの調整**:
   - 「単純化」という言葉を「均衡（バランス）」や「調停」という概念にシフト。
   - 不要な贅肉を削ぎ落とすだけでなく、必要な機能が欠落していないかを確認する「バランサー」としての振る舞いを強化。
4. **依存箇所の修正**:
   - `gemini-extension.json` や `package.json` などで `simplifier` を参照している箇所があれば更新。

## ゴール
- `balancer` エージェントが正常に機能し、`available_subagents` で確認できること。
- 名称変更により、タイピングの負担が軽減されること。
