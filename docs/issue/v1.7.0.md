# Issue: issue_crafter スキルの追加

## 概要
Issue 文書を精緻化し、実用的な実装計画を自動生成・統合するための `issue_crafter` スキルを新規追加する。

## 背景と問題点
Issue 文書の作成において、人間の要望を正確に技術仕様（実装計画）に落とし込む作業には多くの工数がかかる。また、提案された実装計画が実際のコードベースと整合しているかを検証するステップが不足している。

## 解決策
以下のフローを自律的に実行する `issue_crafter` スキルを導入する。
1. ユーザーの要望から Issue 草案を作成。
2. `codebase_investigator` で妥当性を評価。
3. `plan` モードで詳細な実装計画を立案。
4. 計画を Issue 文書に統合。

## 実装内容詳細
- `skills/issue_crafter/SKILL.md` の作成。
- スキルの役割（System Prompt）の定義：
    - `docs/issue/TEMPLATE.md` の遵守。
    - `codebase_investigator` の積極的な活用。
    - `plan` モードの指示。
    - Issue 文書の構造的な更新（`write_file` / `replace` の使用）。

## ゴール
- `skills/issue_crafter/SKILL.md` が配置されていること。
- ユーザーが `activate_skill issue_crafter` でこのスキルを呼び出し、一連のフロー（草案作成から計画統合まで）をシームレスに実行できること。
