# Issue: Jules 連携コマンドの追加

## 概要
Jules エージェントとの連携を強化し、実装依頼から PR 作成までを自動化するカスタムコマンド `jules` を追加する。

## 背景と問題点
現在、Jules を利用した開発プロセス（セッション開始、計画承認、実装待機、検証、PR 作成）は手動の手順が多く、効率化の余地がある。特に、実装されたコードのローカル検証やレビュープロセスを自動化し、品質を担保しつつ工数を削減したい。

## 解決策
`.gemini/commands/jules.toml` を作成し、以下のワークフローを定義する。

1.  **準備**: 現在の変更をコミット・プッシュ。
2.  **依頼**: `jules-client` スキルを用いてセッションを開始し、実装計画を待機。
3.  **審査**: 提案された計画をユーザーが承認（または修正指示）。
4.  **検証**: 実装完了後、成果物ブランチをチェックアウトし、`docs/REVIEW.md` に基づく自動レビューを実行。
5.  **修正**: 軽微な不備は Gemini が修正、重大な不備は Jules に差し戻し。
6.  **完了**: 合格した場合、GitHub PR を自動作成し、元の環境に復帰。

## 実装内容詳細
- `.gemini/commands/jules.toml` の作成。
- プロンプト内での `jules-client` スキルおよび `gh` CLI の活用定義。
- エラーハンドリングと分岐処理の記述。

## ゴール
- `jules <issue_id>` コマンドを実行することで、上記ワークフローが開始されること。
- `.gemini/commands/jules.toml` がリポジトリに含まれていること。
