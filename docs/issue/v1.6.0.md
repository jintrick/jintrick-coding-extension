# Issue: マニフェストからの不要な Hooks 定義の完全削除

## 概要
`gemini-extension.json` に残存している `hooks` フィールドを削除し、マニフェストを `hooks/hooks.json` のみに依存する正しい構成へ修正する。

## 背景と問題点
- **二重定義と不整合**: 現在のマニフェストには `hooks` 配列が含まれており、削除済みの `dist/hooks/open_file_hook.cjs` を参照しているなど、実態と乖離している。
- **仕様準拠**: Gemini CLI の公式ドキュメントでは `hooks/hooks.json` を使用すべきとされており、マニフェストへの直接記述は推奨されていない（または無効である）。

## 解決策
- `gemini-extension.json` から `hooks` フィールド（配列全体）を削除する。

## 実装内容詳細
- `gemini-extension.json` を編集し、`hooks: [...]` ブロックを削除する。

## ゴール
- [ ] `gemini-extension.json` がクリーンになり、`hooks` フィールドが存在しない状態になる。
- [ ] 拡張機能が正常に読み込まれ、`hooks/hooks.json` 経由でフックが動作する。
