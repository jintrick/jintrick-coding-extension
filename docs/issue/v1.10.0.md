# Issue: スキル `prompt_crafter` の新規作成

## 概要
Gemini のプロンプト生成能力の低さを補い、高品質なプロンプト（エージェント仕様）を安定して生成するための専用スキル `prompt_crafter` を導入する。`docs/reference/best-prompt.md` をバイブルとし、草案作成と厳格な推敲の 2 ステップを強制する。

## 背景と問題点
Gemini はプロンプト作成において、抽象的な表現や不明確な完了条件、冗長な説明を多用する傾向がある。これらはエージェントの動作を不安定にし、ノイズとなる。高品質なプロンプトには、徹底した「成果物拘束」と「抽象語の排除」が必要である。

## 解決策
`skills/prompt_crafter/SKILL.md` を作成し、以下の 2 モード・ワークフローを定義する。
1.  **Draft Mode**: 要求を構造化し、必須セクションと出力形式を定義した初稿を作成する。
2.  **Refine Mode**: `docs/reference/best-prompt.md` の設計原則（14 項目）に基づき、草案を検閲・修正する。特に抽象語の排除と判定可能な完了条件の策定を徹底する。

## 実装内容詳細
- `skills/prompt_crafter/SKILL.md` の作成。
- `best-prompt.md` の内容をスキル内の「推敲プロセス」にチェックリストとして組み込む。
- ワークフローの各ステップで出力すべき具体的なフォーマットを定義する。

## ゴール
- `prompt_crafter` スキルが定義され、`activate_skill` で呼び出し可能になること。
- 生成されたプロンプトが `best-prompt.md` の基準をすべて満たしていること。
- 人間による修正を最小限に抑えられるレベルの高品質なエージェント仕様が出力されること。
