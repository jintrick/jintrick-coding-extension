# Issue: デプロイ不備の修正

## 概要
拡張機能のインストール後、`gemini-cli-expert` などのスキルが正しく認識されていない。
調査の結果、一部のスキルディレクトリが入れ子構造（`skills/<name>/<name>/SKILL.md`）になっていることが原因であることが判明した。

## 課題
- `jules-client` と `skill-installer` が入れ子構造になっているため、CLI から認識されない。
- `.skill` ファイル（バイナリ/Zip）が残存しており、不要である。

## 解決策
- `skills/` 配下のすべてのスキルについて、ディレクトリ構造をフラット化し、`skills/<skill-name>/SKILL.md` という配置にする。
- 入れ子の原因となっているサブディレクトリからファイルを引き上げ、サブディレクトリを削除する。
- 不要な `.skill` ファイルも削除する。

## 実装内容詳細
### 1. `jules-client` の修正
- `skills/jules-client/jules-client/` の中身を `skills/jules-client/` に移動する。
- `skills/jules-client/jules-client/` ディレクトリと `jules-client.skill` を削除する。

### 2. `skill-installer` の修正
- `skills/skill-installer/skill-installer/` の中身を `skills/skill-installer/` に移動する。
- `skills/skill-installer/skill-installer/` ディレクトリと `skill-installer.skill` を削除する。

### 3. 検証
- すべてのスキルが `skills/<skill-name>/SKILL.md` の構造になっているか確認する。
