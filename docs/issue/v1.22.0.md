# Issue: 実利的な設計者 (`simplifier`) サブエージェントの導入

## 概要
実装計画に対し、オーバーエンジニアリングや過剰なエッジケースへの対応を抑制し、YAGNI原則（You Ain't Gonna Need It）に基づいた実用的で「バランスの取れた」計画へと修正・助言を行うサブエージェント `simplifier` を追加する。

## 背景と問題点
AIによる自動設計は「念のため」の複雑な抽象化や、低確率なエッジケースへの過剰対応を提案しがちであり、これが保守コストを増大させる。本プロジェクトの「懐疑的なシニアアーキテクト」というペルソナを、計画レビューに特化させたサブエージェントとして切り出し、計画の「贅肉」を削ぎ落とす必要がある。

## 解決策
`agents/simplifier.md` を新規作成する。
- **YAGNIの徹底**: 不要な機能を徹底的に排除し、最小限の構成を提案させる。
- **批判的検証**: 複雑な設計に対し、「それは本当に今必要か？」という問いを投げかけ、シンプルな代替案を提示する。
- **トレードオフの明示**: 複雑化による負債と、簡素化による利益を明確に比較させる。

## 実装内容詳細
1. **`agents/simplifier.md` の作成**
    - `name`: `simplifier`
    - `description`: 計画のオーバーエンジニアリングを抑制し、YAGNIに基づいた実利的な代替案を提示する。
    - `tools`: `read_file`, `grep_search`, `activate_skill`
    - `system prompt`: `GEMINI.md` のペルソナをベースに、計画の「削ぎ落とし」に特化した指示（抽象化の禁止、最小構成の追求）を記述。
2. **検証**
    - 複雑なIssueを読み込ませ、簡素化の提案が行われるかを確認する。

## ゴール
- `simplifier` が正常にロードされ、`available_subagents` に表示されること。
- 計画書に対し、実利的な改善案を提示できること。
